# SPEC-CODIGO (Plantilla minima)

**Nota:** Esta plantilla fue creada porque no existia en el repositorio.
**Objetivo:** Establecer reglas basicas para cambios de codigo.

## Reglas
- Mantener cambios acotados y claros.
- Evitar cambios colaterales no solicitados.
- Preferir funciones puras y mapeos explicitos.
- Agregar comentarios solo cuando la intencion no sea obvia.
- Respetar convenciones existentes del proyecto.

## Trazabilidad
- Todo codigo modificado debe incluir comentario JSDoc con ID de intervencion y ruta del documento de respaldo.
